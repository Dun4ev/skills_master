# Мозговой штурм идей (Brainstorming Ideas)

## Когда использовать этот навык
- Перед любой творческой работой (создание фич, компонентов).
- Для изучения намерений пользователя и требований.
- Для превращения сырых идей в четкий дизайн/спецификацию.
- Когда пользователь говорит "Давай обсудим" или "У меня есть идея".

## Рабочий процесс (Workflow)
- [ ] **Понимание идеи**: Изучите контекст, задавайте уточняющие вопросы (по одному!).
- [ ] **Исследование подходов**: Предложите 2-3 варианта с плюсами/минусами.
- [ ] **Презентация дизайна**: Представляйте дизайн частями (200-300 слов), валидируя каждый блок.
- [ ] **Документация**: Сохраните итог в `docs/plans/YYYY-MM-DD-<topic>-design.md`.

## Инструкции

### 1. Понимание (Understanding)
- Сначала проверьте текущее состояние проекта (файлы, доки).
- Задавайте вопросы **по одному за раз**.
- Предпочитайте вопросы с вариантами ответов (multiple choice).
- Фокусируйтесь на цели, ограничениях и критериях успеха.

### 2. Исследование (Exploring)
- Всегда предлагайте альтернативы (2-3 подхода).
- Объясняйте свой выбор (почему вариант А лучше варианта Б).

### 3. Презентация (Presenting)
- Разбивайте дизайн на секции (Архитектура, Данные, UI, Тесты).
- Спрашивайте "Все ли верно?" после каждого блока.
- Будьте готовы вернуться и уточнить.

### 4. После дизайна (After the Design)
- Запишите валидированный дизайн в файл.
- Спросите: "Готовы перейти к реализации?"
- Если да -> предложите использовать навык `planning-work`.

## Ключевые принципы
- **Один вопрос за раз** — Не перегружайте пользователя.
- **YAGNI** — Безжалостно удаляйте лишнее.
- **Инкрементальная валидация** — Согласовывайте дизайн по частям.
- **Гибкость** — Не бойтесь менять подход в процессе обсуждения.
